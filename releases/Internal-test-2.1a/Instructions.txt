PUBLIC BETA v2.0 - INSTALLATION INSTRUCTIONS
============================================

READ THIS FILE COMPLETELY BEFORE STARTING.


1. RELEASE IDENTITY AND RISK
----------------------------

This is Public Beta v2.0 of the independent Legion Go 1 Graphics Driver
Toolkit. It is not produced, endorsed, or supported by Lenovo, AMD, or
Microsoft.

Public Beta v2.0 upgrades the workflow from the earlier AMD 26.6.2 release to
the validated AMD 26.6.4 target while substantially broadening compatibility
with healthy starting configurations and legitimate package variations.

Compatibility in this release means that the workflow no longer depends on
one exact starting display-driver version, one exact Lenovo extension version
or hash, one exact AMDUWP package identity, one exact outer AMD installer
filename or hash, or one exact Windows SDK/WDK build. The scripts validate the
required hardware, signatures, structure, semantics, active attachment, and
extracted target payload instead.

Public Beta v2.0 still validates and installs the exact AMD 26.6.4 target used
for this release. Do not substitute another AMD driver version merely by
renaming its installer.

The complete workflow passed from a fresh Lenovo OEM graphics installation on
an original Lenovo Legion Go on July 10, 2026. The final persistence audit
reported:

SCRIPT 4 PASS: True
Failed checks: 0
TOOLKIT COMPLETE: True

The scripts modify the display-driver package, Driver Store, local certificate
trust, catalog registration, AMD Software, AppX registration and provisioning,
AMD/Lenovo compatibility metadata, and temporary Windows Test Signing state.

Use the toolkit at your own risk. Back up important files first. Keep the
Legion Go connected to power. If Windows Device Encryption or BitLocker is
enabled, make sure the recovery key is available before changing Secure Boot
or boot configuration.


2. SUPPORTED CONFIGURATION
--------------------------

Public Beta v2.0 supports:

- The original Lenovo Legion Go, referred to in this project as Legion Go 1.
- GPU hardware identity:
  PCI\VEN_1002&DEV_15BF&SUBSYS_381217AA
- 64-bit Windows 11, build 22000 or newer.
- A healthy AMD display stack on the supported GPU. This may be the Lenovo OEM
  driver, a prior validated toolkit installation, or the 26.6.4 target during
  recovery.
- A healthy Microsoft hardware-signed Lenovo Legion Go extension that satisfies
  the required hardware targeting, ExtensionId, registry directives,
  declarative package structure, and coherent Lenovo metadata contract.
- A healthy, structurally compatible Microsoft-signed AMD-UWP Version Control
  component.
- At least 12 GB free on the C: drive or selected workspace volume.
- One official AMD-signed Windows 11 installer reporting version 26.6.4.0 and
  containing the exact validated target payload.

Do not use this toolkit on another Legion model, another AMD device, or a
different GPU hardware identity. Script 1 performs its own hardware and
platform checks and must pass before any later script is run.

5. BEFORE RUNNING SCRIPT 1
--------------------------

1. Start from a healthy AMD display stack on the supported original Legion Go.
   Script 1 records the active version, INF, kernel, hashes, and compatible
   Lenovo extension dynamically.

2. Save all work and close unnecessary applications.

3. Connect the Legion Go to AC power.

4. Keep all toolkit files together. Do not rename, move, replace, or edit them
   while the workflow is in progress.

5. Do not run DDU as part of this procedure.

6. Do not manually install another AMD driver or AMD Software package during
   the workflow.

7. Do not manually enable or disable Test Signing between script runs.

8. Do not delete these working or state folders while the workflow is active:

   C:\AMD\LegionGo-26.6.4
   C:\ProgramData\LegionGo-AMD-26.6.4

9. Open Windows PowerShell as Administrator. Use Windows PowerShell 5.1 as the
   launcher. Script 1 invokes PowerShell 7 internally where required.

10. Do not use the script right-click action "Run with PowerShell." Launch each
    script with the exact command in this guide.

The -ExecutionPolicy Bypass option below applies only to that PowerShell
process. It does not permanently change the computer's execution policy.


6. PREREQUISITES AND INTERNET USE
---------------------------------

Script 1 checks for:

- PowerShell 7.4 or newer.
- 7-Zip command-line tools.
- A functional paired x86 Inf2Cat and x64 SignTool installation. The workflow
  accepts the installed kit by capability rather than one mandatory kit build.
- The supported Windows and Legion Go hardware.
- At least 12 GB free space.

When a required tool is missing, Script 1 lists the exact WinGet packages and
asks for permission before installing anything:

- Microsoft.PowerShell
- 7zip.7zip
- Microsoft.WindowsSDK.10.0.28000
- Microsoft.WindowsWDK.10.0.28000

Those SDK/WDK packages are automatic-install fallbacks. An already installed
functional paired kit may use another version.

Internet access may be needed only for approved missing-prerequisite
installation through WinGet. The toolkit does not download the AMD installer
and does not accept AMD's license agreement on the user's behalf.

With Secure Boot already disabled:

8. SCRIPT 1 - PREPARE AND SIGN THE DRIVER PACKAGE
--------------------------------------------------

Run:

PowerShell.exe -NoProfile -ExecutionPolicy Bypass -File "$env:USERPROFILE\Downloads\LeGo-toolkit\01-Prepare-Build-Sign-And-Enter-Test-Signing.ps1"

Script 1 validates the platform and source, records the healthy starting state,
reconstructs the corrected package, creates a per-installation certificate and
catalog, prepares temporary Test Signing, and asks before restarting.

After Windows restarts, run Script 1 again with the same command.

Do not run Script 2 until Script 1 reports:

Ready for Script 2: True


9. SCRIPT 2 - INSTALL THE DRIVER AND RETURN TO NORMAL SIGNING
-------------------------------------------------------------

Run:

PowerShell.exe -NoProfile -ExecutionPolicy Bypass -File "$env:USERPROFILE\Downloads\LeGo-toolkit\02-Install-Driver-And-Verify-Normal-Signing.ps1"

Script 2 verifies the saved and live state, asks before installing and binding
the corrected driver, registers AMD's official Microsoft-signed catalog,
configures Test Signing off for the next boot, and asks before restarting.

The display may flash, reset, or briefly go dark while the driver is bound.

After Windows restarts, run Script 2 again with the same command. The second run
verifies the driver, INF, loaded kernel, GPU health, catalog state, Lenovo
extension, AMDUWP, current boot, and normal Windows signing state.

Do not run Script 3 until Script 2 reports:

SCRIPT 2 PASS: True
Ready for Script 3: True


10. SCRIPT 3 - INSTALL AND VALIDATE AMD SOFTWARE
------------------------------------------------

Run:

PowerShell.exe -NoProfile -ExecutionPolicy Bypass -File "$env:USERPROFILE\Downloads\LeGo-toolkit\03-Install-AMD-Software-And-Reboot.ps1"

Script 3 revalidates the recorded AMD container, extracts the exact native
AMD Software .2099 package, installs or repairs it, validates native RSXCM
22.10.0.0, retires conflicting legacy .2089 MSI and Store AppX state, preserves
compatible host components, restores Lenovo metadata, and opens AMD Software
for confirmation.

When prompted, type YES only after confirming both:

1. AMD Software shows the normal dashboard.
2. The Windows desktop right-click menu shows exactly one
   "AMD: Radeon Software" entry.

After the requested restart, run Script 3 again with the same command.

Do not run Script 4 until Script 3 reports:

SCRIPT 3 PASS: True
Ready for Script 4: True

Run:

PowerShell.exe -NoProfile -ExecutionPolicy Bypass -File "$env:USERPROFILE\Downloads\LeGo-toolkit\04-Final-Persistence-Audit.ps1"

Script 4 is the final read-only audit. It verifies the complete post-restart
state, including the driver, INF, loaded kernel, GPU health, catalogs, signing
state, Lenovo extension, AMDUWP, AMD Software, RSXCM, legacy package absence,
metadata, DxDiag, and relevant event logs.

AMD Software opens again for final visual confirmation. Type YES only after
confirming the normal dashboard and exactly one desktop context-menu entry.

The workflow is complete only when Script 4 reports:

SCRIPT 4 PASS: True
Failed checks: 0
TOOLKIT COMPLETE: True

Script 4 writes:

C:\ProgramData\LegionGo-AMD-26.6.4\final-audit-result.json
C:\Users\<YOUR USERNAME>\Desktop\LegionGo-AMD-26.6.4-Final-Report.txt

Script 4 is read-only and may be rerun later as a persistence check.

14. SECURE BOOT AFTER COMPLETION
--------------------------------

Secure Boot must be disabled while Test Signing is activated for the temporary
installation phase. Script 2 turns Test Signing off, but the toolkit does not
automatically re-enable Secure Boot.

For the simplest validated workflow, leave Secure Boot unchanged until all four
scripts report success. Secure Boot may then be re-enabled manually if desired.
After doing so, start Windows and rerun Script 4. Script 4 accepts Secure Boot
either off or on, but Test Signing must remain off.

16. WHAT CHANGED IN PUBLIC BETA v2.0
------------------------------------

Compared with Public Beta v1.1, Public Beta v2.0:

- Moves the validated target from AMD 26.6.2 to AMD 26.6.4.
- Accepts a healthy compatible starting display stack instead of one exact OEM
  version.
- Validates the active Lenovo extension semantically rather than requiring one
  exact version and hash.
- Accepts a healthy structurally compatible Microsoft-signed AMDUWP package.
- Accepts legitimate AMD-signed 26.6.4 installer container variations only
  after proving the exact extracted target payload.
- Treats the total extracted source-tree file count as telemetry while keeping
  the canonical dependency manifest exact.
- Discovers a functional paired Inf2Cat and SignTool installation by capability
  instead of requiring one exact Windows Kit version.
- Preserves strict Legion Go 1 hardware gating, signature checks, payload
  identities, reboot boundaries, and final persistence verification.
